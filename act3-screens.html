Act 3 — Project Memory

6 screens · Carbon & Amber theme · March 31 2026

Once you're working seriously in Amplifier, sessions multiply. Some are running, some are done, some you vaguely remember starting two days ago. The CLI shows you session_a3f2b1 · 2 days ago. That's not memory. That's an ID.

Canvas solves this at two levels. Session memory: what did this session do, what did I ask, what changed? Project memory: what is this project, loaded into every new session automatically. Together, every session stops being an island.

A — Session Recall
Step 9
Session Hub
Three sessions running in the same project — "Auth middleware" (28m), "Fix test failures" (4m), "Refactor payments" (paused). Each card shows the session title derived from the first prompt, current status, and a one-line preview of the last thing it did. A list of intentions, not IDs.
Step 10
Session Completes
"Auth middleware" finishes. It dims in the session panel — still visible, not gone. A summary card expands: what it did in one sentence, the last prompt the user sent, 12 files changed. An Archive button sits quietly at the bottom. It'll auto-archive in 24 hours either way.
Step 11
Archive — Session History
User opens the archive. Eight past sessions, each showing title and a one-sentence summary. "Auth middleware — implemented JWT auth with refresh token rotation." "Fix test failures — resolved 14 failing tests in payments module." Scan in 30 seconds. This is what amplifier session resume should feel like.
B — Parallel Work
Step 12
Parallel Worktrees
User starts a new session while two others are still running. Canvas automatically routes it to a separate git worktree. A subtle indicator on each session card shows the worktree path. Both sessions read and write files independently — no conflicts, no coordination needed.
C — Project Memory
Step 13
Project Context
The project pane surfaces Canvas's understanding of the project: what it is, what the current goal is, recent session titles. Auto-inferred from git history and session activity. A pencil icon — it's the user's to edit. This replaces re-reading VISION.md every session. The LLM reads this instead.
Step 14
New Session, Pre-loaded
A new session starts. The prompt area shows a context pill: "Project context loaded — Canvas-App." No VISION.md re-read. No "let me understand your codebase." The session already knows what you're building, what's been done, and what's in progress. The session starts informed.
